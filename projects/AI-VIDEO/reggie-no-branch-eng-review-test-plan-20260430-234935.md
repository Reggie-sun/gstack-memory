## Affected Pages/Routes

- CLI `ai-video validate` — validate project config, shot list, workflow template, binding paths, and local-only ComfyUI URL safety.
- CLI `ai-video run` — execute dry-run and mocked generation flow, create manifest, render workflows, chain last frames, and produce final artifact path.
- CLI `ai-video resume` — resume interrupted runs, skip valid completed shots, re-extract missing last frames, and rerun stale downstream shots.

## Key Interactions to Verify

- Run validation against a correct example project and confirm exit code 0.
- Run validation against UI workflow JSON and confirm a clear API-format error.
- Run a mocked 3-shot chain and confirm shot 2 receives shot 1 last frame, shot 3 receives shot 2 last frame, and final stitching is requested after all shots succeed.
- Resume a manifest where one completed shot is valid, one clip lacks a last frame, and one downstream shot is stale.
- Exercise ComfyUI client responses for validation error, queue timeout, missing history, execution failure, missing output node, multiple output candidates, and successful video output.

## Edge Cases

- Non-local `comfy.base_url` without explicit opt-in.
- Binding path missing a key or pointing through a wrong container type.
- Character ID referenced by a shot but missing from project config.
- Reference image path exists but content hash differs from the manifest on resume.
- ffmpeg paths containing spaces in clip filenames.
- ffmpeg or ffprobe unavailable locally.

## Critical Paths

- Validate config -> render API-format workflow snapshot -> submit mocked Comfy job -> collect video artifact -> ffmpeg extract last frame -> pass frame to next shot -> normalize clips -> concat final MP4.
- Interrupted run -> load manifest -> verify hashes -> skip valid shots -> recover missing last frame -> rerun stale downstream shots.
