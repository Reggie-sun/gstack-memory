## Vision (aspirational, not in scope)

### 10x Check
The 10x version would be a self-tuning retrieval system: query intent classified automatically, chunk weighting adjusted dynamically based on real-time eval feedback, corpus quality continuously monitored with automatic drift detection and regression alerts. Instead of hand-crafted canary conditions, a lightweight learned reranker would handle the hierarchy promotion logic. The eval infrastructure would run as CI on every retrieval change, with frozen baselines automatically relocked when sample sets change.

### Delight Opportunities (aspirational, P2/P3 priority)
1. (P2) Corpus drift regression test that alerts when new docs shift baseline
2. (P1) Canary interaction matrix: a single test that verifies all canaries compose correctly
3. (P2) Eval result diff tool that highlights per-sample changes between runs
4. (P3) Retrieval debug UI showing why each canary fired for a given query
5. (P3) Auto-canary suggestion: given a failing eval sample, suggest the narrowest canary that fixes it
