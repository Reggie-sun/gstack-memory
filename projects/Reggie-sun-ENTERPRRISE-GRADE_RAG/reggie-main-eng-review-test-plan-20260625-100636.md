## Affected Pages/Routes
- Backend chat deterministic named-document overview answer path — verify document overview questions return the right answer shape and selected citations.
- Backend answer format policy — verify new overview policy titles and prompt guidance are available without changing normal summary resolution.

## Key Interactions to Verify
- Named project technical agreement overview returns project-specific sections and keeps selected citations.
- Named operation manual overview returns workflow/module sections while filtering UI noise.
- Named standard overview returns requirement/method/check sections while preserving revision metadata filtering.
- Named SOP/WI overview without parsed text returns work-instruction sections.
- Named SOP/WI or installation manual overview with parsed outline/details preserves parsed detail output instead of switching to snippet-only templates.

## Edge Cases
- No concrete family-specific evidence slots are available: renderer should fall back to generic summary instead of showing a mostly empty specific template.
- Noisy UI snippets contain operation keywords: answer must not surface low-signal UI actions such as deletion/export button text.
- Existing maintenance manual overview policy still returns its current maintenance-specific sections.
- Evidence probe and contextual detail follow-up questions must not be intercepted by overview rendering.

## Critical Paths
- `try_build_named_document_overview_answer()` selection and rendering.
- Route-gated `document_overview` path through `try_build_route_gated_document_answer()`.
- Direct deterministic citation answer fallback through `try_build_deterministic_citation_answer()`.
- Focused verification: backend named overview tests plus `make verify-grounding-answer-policy-gate`.
