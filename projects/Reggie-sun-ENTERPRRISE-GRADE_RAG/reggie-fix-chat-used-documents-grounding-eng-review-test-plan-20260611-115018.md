## Edge Cases

- Multiple formal SOP candidates in the top results: no multi-document fan-out; answer must not mix neighboring SOP procedures.
- Formal SOP evidence missing or weak: system should fail closed or clarify, not invent executable steps.
- LLM response declares wrong `used_documents.document_id`: title plus context match may recover, wrong id plus wrong title must still fail closed.
- Streaming strict mode: answer deltas must not show unsafe pre-repair content before grounding and SOP shape repair finish.

## Critical Paths

- Helper/unit path: short-topic bridge detection, strong-action detection, lookup/definition negatives.
- Sync path: `皮带装配` returns the agreed response shape and keeps citations scoped to one formal SOP document.
- Streaming path: `皮带装配` reaches the same final mode/answer/source behavior as sync.
- Live API path: four smoke accounts return consistent behavior and preserve department/account visibility.
- Browser path: headed `/portal` replay shows the final answer, citations/source cards, and no stale no-context UI state.
