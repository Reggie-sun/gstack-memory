## Affected Pages/Routes
- `/api/v1/sops/generate/document` — verify document preview fallback still works when retrieval returns no citations, and verify preview data is consumed exactly once after the refactor.
- `/api/v1/sops/generate/document/stream` — verify SSE `done` payload still builds dossier/final page from preview text, and verify retryable LLM failure still degrades cleanly to retrieval fallback in stream mode.
- `/api/v1/sops/generate/topic` — verify service-to-orchestrator request shaping is unchanged by the new resolver seam.
- `/api/v1/sops/generate/topic/stream` — verify stream delegation and failure semantics are unchanged when the new resolver is wired in.

## Key Interactions to Verify
- Generate SOP from a document with empty retrieval results but non-empty preview text.
- Stream-generate SOP from a document and inspect `meta` / `content_delta` / `done` ordering.
- Trigger retryable LLM failure in both sync and stream orchestrator paths and confirm degraded citations + retrieval fallback draft are used.
- Inspect resolver unit behavior for preview splitting, dossier source resolution, and fallback draft formatting.

## Edge Cases
- Document preview is empty.
- Document preview exists but contains more than three logical chunks.
- Preview snippet exceeds truncation threshold.
- Document mode has empty preview text but non-empty citations.
- Runtime gate rejects the request while using the new resolver seam.

## Critical Paths
- Document request -> citation pipeline empty -> preview fallback citations -> generation/finalization succeeds.
- Document stream request -> preview text feeds dossier final page -> SSE done payload remains complete.
- Retryable LLM failure -> degraded citations -> retrieval fallback draft -> telemetry success path remains correct.
