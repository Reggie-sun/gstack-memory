## Key Interactions to Verify
- Natural-language repo request like `帮我修这个接口` enters `runtime-workflow` with `resolve` mode.
- Explicit specialized entries like `/plan-eng-review` and `/investigate` bypass passive runtime.
- Explicit lanes like `flow-api` and repo commands like `/plan` bypass passive runtime.
- General prompts like `帮我写一封请假邮件` are rejected even when IDE context exists.
- Direct-edit prompts like `只改这个文件里的一个文案` do not get over-admitted once the direct-edit hardening lands.

## Edge Cases
- Slash-prefixed repo path text like `/portal 登录页布局错了，帮我修一下` should still count as repo work, not as an explicit command.
- Missing bridge service should fail loudly in CLI smoke instead of silently pretending runtime succeeded.
- Prompt-only direct-edit requests without `directEditTarget` should bypass passive runtime after the recommended hardening.

## Critical Paths
- User prompt -> eligibility gate -> runtime trace -> `resolve-workflow` bridge call.
- User prompt -> eligibility gate bypass -> host handles explicit skill / command / direct edit itself.
- `resolve` result with `auto_execute_recommended` -> `advance-workflow` follow-up call.
