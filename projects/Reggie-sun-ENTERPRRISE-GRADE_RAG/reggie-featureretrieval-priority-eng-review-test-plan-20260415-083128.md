## Affected Pages/Routes
- CLI `scripts/ocr_engine_spike.py` — verify the spike harness accepts explicit model inputs, stays read-only, and emits comparable dry-run output for `1591` and `3639`.

## Key Interactions to Verify
- Run baseline `mobile/mobile` dry-run for `1591` and `3639` and confirm output includes comparable metrics (`page_score`, `candidate_count`, `table_block_count`, `table_chunk_count`, `table_headers_preview`).
- Run candidate model combos and confirm baseline/candidate outputs are isolated rather than reusing the same Paddle singleton instance.
- Confirm spike dry-run never calls embedding or vector write paths.
- Confirm the one-off canary smoke path uses explicit `Settings(...)` override in a single process rather than mutating the long-lived worker runtime.

## Edge Cases
- Invalid or missing document selector / artifact path should fail with an actionable error.
- Invalid det/rec model combination should fail loudly rather than silently falling back.
- Missing OCR dependencies should surface the existing `OCRUnavailableError` path cleanly.
- Candidate path that improves page preview but regresses document-level table structure should be reported as rejected, not promoted.

## Critical Paths
- `OCRClient._get_paddle_ocr()` default mobile baseline remains unchanged.
- `OCRClient._get_paddle_ocr()` creates distinct singleton instances for different det/rec model combinations.
- `scripts/ocr_engine_spike.py` builds parsed-document + chunk results only, without embedding/vector writes.
- Document-level dry-run comparison for `1591` and `3639` is reproducible and sufficient to make a go/no-go decision before any promotion work.
