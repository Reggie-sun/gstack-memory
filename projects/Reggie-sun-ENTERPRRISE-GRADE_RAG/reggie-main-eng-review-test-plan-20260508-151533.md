## Affected Pages/Routes
- `/api/v1/documents` (POST) — Document upload and ingest pipeline
- `/api/v1/documents/{id}` (GET) — Document preview and retrieval
- Chat citation pipeline — Citation building from retrieved chunks

## Key Interactions to Verify
- Table chunk retrieval: user queries table content → system returns table_retrieval chunk → citation shows table_display chunk structure
- Dual chunk dedupe: same table should not appear twice in top-k results
- PostProcessor compat layer: end-to-end ingestion produces identical results to current pipeline
- StructureProfile selection: GB/T standard → gbt_standard profile; unknown doc → generic profile
- Screening split: mechanical noise tags read from ExtractedDocument, semantic judgment preserved

## Edge Cases
- Multi-page table continuation (continued_from/continued_to chain breaks)
- Document with no matching StructureProfile (fallback to generic)
- PostProcessor failure (all-or-nothing rollback to raw pipeline)
- Large PDF (100+ pages) memory pressure during PostProcessor processing
- Non-Chinese/English documents with unrecognized noise patterns
- Table with both landscape and portrait orientation
- OCR text with very low confidence scores

## Critical Paths
- Upload PDF → Parse → PostProcessor → Compat layer → Screen → Chunk → Embed → Index → Verify identical results
- Upload GB/T standard → PostProcessor → StructureClassifier → gbt_standard profile → StructuredDocument → Dual chunk → Embed → Index
- Query table content → Retrieve table_retrieval → Dedupe → Citation shows table_display
