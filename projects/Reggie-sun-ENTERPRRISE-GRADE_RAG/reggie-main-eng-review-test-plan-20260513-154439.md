## Affected Pages/Routes
- `/api/v1/retrieval/search` — verify retrieval diagnostics, supplemental behavior, and post-refactor result semantics remain stable.
- `/api/v1/chat/ask` — verify `fast` and `accurate` modes preserve grounded citations, especially with session memory and external fallback gates.
- `/api/v1/chat/ask/stream` — verify streamed meta/done payloads stay aligned with non-stream behavior after grounding/fallback changes.
- `/api/v1/documents`, `/api/v1/ingest/jobs/{job_id}` — verify upload/ingest contract remains stable when later P1 golden-flow audits reuse existing smoke coverage.
- `portal` document center / library / chat surfaces — verify current regression suites still cover upload visibility, preview, citation rendering, and document-state transitions before adding any new “golden flow” tests.

## Key Interactions to Verify
- Follow-up question with existing session memory in `fast` mode keeps citations grounded to current retrieved context.
- Follow-up question with no useful session memory does not inherit stale topic/citation state.
- Retrieval eval run surfaces a machine-readable comparability verdict when ACL seed, active snapshot, or governance preconditions are missing.
- Portal library upload -> ingest -> preview -> chat source opening still works through the existing regression entrypoints before any suite expansion.

## Edge Cases
- `fast + session` returns an answer that appears correct but cites the wrong internal document.
- `enable_external_search=false` with low-quality internal citations does not silently escape to external fallback.
- Eval report shows a strong score while the active metadata store has not been ACL-seeded for evaluation.
- Snapshot/governance alignment reports classify the same case differently between checked-in and runtime-backed inputs.
- Streamed chat meta payload and final done payload disagree on mode/citations after fallback gating.

## Critical Paths
- Retrieval comparability gate: sample loading -> auth profile mapping -> ACL-seeded metadata assumptions -> governance/snapshot alignment -> final verdict in eval output.
- Fast-memory grounding gate: session memory rewrite -> retrieval/citation build -> grounding filter -> fallback decision -> visible citations in response.
- Existing golden regression audit: main chain smoke + frontend regression runner -> identify missing user-visible gaps -> add only targeted coverage, not a parallel suite.
