## Affected Pages/Routes
- `/api/dashboard` — New reverse arbitrage signals and Steam sell positions should appear in dashboard
- Steam Community Market — sellitem API calls for automated selling
- BUFF163 market — existing buy flow now also handles REVERSE_STEAM_ARBITRAGE signals

## Key Interactions to Verify
- Reverse arbitrage scoring: BUFF price vs Steam price ratio calculation on dashboard
- Steam sell execution: item appears in Steam inventory → sellitem called → listing created
- Position routing: REVERSE_STEAM_ARBITRAGE positions route to SteamSellExecutor, not BuffOrderExecutor
- Config toggle: reverse_steam_arbitrage.enabled=false → no reverse signals generated

## Edge Cases
- Steam API returns 429 during order book fetch → graceful skip, no crash
- No matching Steam item for a BUFF item → skip without error
- Steam inventory check finds item not yet arrived (still in 7-day hold) → skip sell
- highest_buy_order is 0 or missing → fall back to limit order mode
- Price changes between scan and sell → re-verify before placing order
- CNY vs USD currency mismatch in price comparison → correct conversion

## Critical Paths
- Full cycle: BUFF item scanned → ratio below threshold → order book fetched → BUY signal → BuffOrderExecutor buy → 7-day hold → SteamSellExecutor sell → profit realized
- Error recovery: Steam API failure → skip this tick → next tick retry from scan
